# House Rules

## 1. 文档目的
本文件定义本项目/本团使用的《克苏鲁的呼唤（CoC）7版》自定义规则、裁定偏好与实现约束。

目标：
- 明确哪些地方沿用官方规则
- 明确哪些地方使用 house rule
- 让程序、AI KP、AI 调查员、真人 KP 对规则优先级有统一理解
- 减少跑团过程中因规则解释不同而产生的冲突

---

## 2. 适用范围
- 适用规则集：CoC 7th Edition
- 适用场景：
  - [x] 所有 campaign
  - [ ] 仅限特定 campaign
  - [ ] 仅限测试模式
- 适用语言：
  - 用户显示语言：简体中文
  - 内部代码/字段命名：英文

---

## 3. 规则优先级
当规则发生冲突时，按以下优先级处理：

1. 当前 campaign 的人工指定裁定
2. 本文档中的 house rules
3. 当前 campaign 指定模组/补充规则
4. CoC 7 主规则书
5. 其他补充资料
6. AI 推断（最低优先级，不可直接视为最终规则）

## 5. 人物创建规则

### 5.1 属性生成方式
- 采用方式：
  - [x] 标准 CoC 7
  - [x] 点购/点买
  - [ ] 固定模板
  - [ ] 自定义
- 具体说明：
  默认采用 CoC 7 标准创建方式。也可以根据KP提供的购点规模来进行购点。测试模式下允许程序快速生成合法人物卡草案，但必须标记为 generated_draft。

### 5.2 可选时代 / 背景
- 默认时代：1980s
- 是否允许跨时代内容：
  - [x] 是
  - [ ] 否
- 限制说明：
  非默认时代必须在 campaign 配置中明确指定。跨时代时，装备、职业和知识背景必须同步调整。

### 5.3 职业与技能点
- 职业点计算方式：
  按官方职业规则执行，优先采用人物设定合理的属性组合。具体算法在人物卡中。
- 兴趣点计算方式：

- 是否允许自定义职业：
  - [x] 是
  - [ ] 否
- 自定义职业限制：
  自定义职业必须有人类可解释的现实背景；必须给出核心技能组；不得为了堆数值而脱离设定。


### 5.4 年龄规则
- 是否使用官方年龄修正：
  - [x] 是
  - [ ] 否
- 若否，自定义规则如下：
   不适用。

### 5.5 人设与背景要求
- 是否要求每张卡必须包含：
  - [x] 信念/信条
  - [x] 重要之人
  - [x] 重要之地
  - [x] 宝贵之物
  - [x] 特质
  - [x] 心理弱点/恐惧

   每张人物卡至少应包含职业、社会背景、核心动机、一个秘密或心理弱点、一个可被剧情利用的人际联系。


### 5.6 车卡自动化限制

  - [x] 是
  - [ ] 否

   背景草稿、技能建议、人物关系建议、装备建议、个性标签建议。
- AI 不可自动决定的部分：

## 6. 检定规则

### 6.1 基础检定
- 困难成功 / 极难成功是否按官方：
  - [x] 是
  - [ ] 否
- 自定义说明：
  默认按官方执行，叙事层可对信息呈现方式进行调整，但不得改动成功等级事实。

### 6.2 奖励骰 / 惩罚骰
- 是否严格按官方：
  - [x] 是
  - [ ] 否
- 特殊裁定：
  合理准备、道具加成、充足时间、玩家优质描述可考虑奖励骰；恐慌、仓促、环境恶劣可考虑惩罚骰。

### 6.3 对抗检定
- 默认处理方式：
  按官方流程处理。程序输出必须明确双方技能值、成功等级、判定依据。

### 6.4 组合检定 / 连续检定
- 是否允许：
  - [x] 是
  - [ ] 否
- 处理方式：
   允许多步骤调查，但程序应避免无意义重复检定。相似意图在短时间内重复尝试应提高代价或风险。

### 6.5 推动检定（Pushed Roll）
- 是否允许：
  - [x] 是
  - [ ] 否
- 允许条件：
  需要叙事上合理且玩家明确说明如何重试。
- 失败后果处理倾向：
  推动失败应带来清晰且可叙事解释的严重后果，如暴露、损坏证据、引发危险、时间损失、误导信息。

## 8. 战斗 / 追逐 / 伤势规则

### 8.1 战斗风格
- [ ] 严格规则
- [ ] 简化处理
- [x] 叙事优先
- 说明：
  默认仍按官方结算伤害与结果，但叙事表现偏紧张、残酷、快速，不鼓励把战斗作为常规问题解决手段。

### 8.2 追逐规则
- 是否启用完整追逐规则：
  - [ ] 是
  - [x] 否
- 若简化，简化方式：
  默认采用简化追逐：分段目标、风险节点、关键检定。复杂追逐可按场次需要切换完整规则。


### 8.3 重伤 / 濒死 / 治疗
- 自定义说明：
   尽量遵循官方，但在剧情关键处允许由 KP 通过代价换取“重伤但未当场退场”的处理。别冷酷无情没几步把人全整死了。

## 9. 线索与调查规则

### 9.1 核心线索政策
- 核心线索是否允许因一次失败完全锁死：
  - [ ] 是
  - [x] 否
- 说明：
  核心线索必须可通过其他方法、额外代价、不同场景或后续调查重新获得。

### 9.2 线索分发方式
- [ ] 严格基于行动
- [x] 允许适度保底
- [x] 叙事优先
- 说明：
  线索发放仍需行动触发，但对于主线推进必需信息，系统应保证存在至少两种以上可达路径。

### 9.3 图书馆使用 / 搜索 / 察觉的裁定偏好
- 说明：
  图书馆与搜寻类检定更偏向“获得更多/更快/更准确”而非“获得或完全得不到”。失败时仍可给较模糊、较慢或带误差的信息。

## 11. 自动化与 AI 行为限制

### 11.1 AI KP
- AI KP 可自动执行的内容：
  普通场景描述草案、NPC 对话草案、线索整理建议、节奏建议、低风险氛围文本。
- AI KP 必须人工确认的内容：
  关键剧情推进、秘密揭示、疯狂表现、重大伤害、死亡、结局走向、长记忆摘要。
- AI KP 不可自动执行的内容：
 擅自改变规则结算、擅自公开私密信息、擅自决定最终结局。


### 11.2 AI 调查员
- AI 调查员可自动建议的内容：
   行动建议、提问建议、调查路径建议、对话草稿、根据人设给出的倾向选择。
- AI 调查员可自动行动的内容：
  仅限测试模式下的低风险动作，且建议默认关闭。
- AI 调查员必须人工审阅的内容：
   所有会改变场景状态、人物关系、秘密信息、长记忆的最终行动。

### 11.3 人工审阅优先级
- 以下内容必须经过人工审阅后才可成为最终记录：
  - [x] 场景推进
  - [x] 重要检定解释
  - [x] 疯狂表现
  - [x] 角色性格漂移
  - [x] 关键线索揭示
  - [x] 结局走向
  - [x] 其他：角色死亡、永久伤害、重大秘密公开

## 14. 当前已确认的 house rules 清单

### Rule HR-001
- 名称：核心线索不可单次失败锁死
- 分类：调查流程
- 说明：主线推进必需的核心线索必须存在替代获取路径
- 优先级：高
- 是否已代码实现：待实现
- 是否需要人工审阅：否
- 示例：图书馆检定失败后，仍可通过访问报社档案、采访知情者或现场搜寻获得相关信息


### Rule HR-002
- 名称：AI 最终动作必须人工确认
- 分类：自动化限制
- 说明：AI KP 和 AI 调查员的最终状态改变性输出必须先经过人工审阅
- 优先级：最高
- 是否已代码实现：待实现
- 是否需要人工审阅：是
- 示例：AI 提议“调查员 A 独自进入地下室”，只有人工确认后才写入正式记录


### Rule HR-003
- 名称：中文优先输出
- 分类：语言策略
- 说明：面向用户的默认文本均使用简体中文，必要时可加英文括注
- 优先级：高
- 是否已代码实现：待实现
- 是否需要人工审阅：否
- 示例：侦查（Spot Hidden）、图书馆使用（Library Use）
