# Review Policy

## 1. 文档目的
本文件定义本项目中“AI 输出在成为正式文本/正式行动/正式状态变化前”的人工审阅与修改机制。

目标：
- 保证 AI 不会直接擅自改变剧情与状态
- 允许主持人/操作者在最终确认前插入、改写、覆盖 AI 输出
- 让系统后续记忆、行为趋势、剧情状态以“人工确认后的版本”为准
- 使项目保持半自动而非失控自动化

---

## 2. 适用对象
本审阅政策适用于以下 AI 输出：

- [x] AI KP 场景叙述
- [x] AI KP 场景推进
- [x] AI KP 结局方向建议
- [x] AI 调查员行动建议
- [x] AI 调查员最终行动文本
- [x] AI NPC 对话
- [x] AI 秘密线索提示
- [x] AI 疯狂表现建议
- [x] AI 人格/行为倾向调整建议
- [ ] 其他：____________

## 3. 核心原则

### 3.1 AI 草案不具权威性
AI 生成的任何文本或行动，在人工确认前都只能视为草案，不可直接写入正式历史。

### 3.2 只有最终确认版本才是正式记录
以下内容只有在人工确认后，才可成为系统的正式历史：
- 场景文本
- 行动文本
- 结构化行动结果
- 状态变化
- 角色行为模式更新
- 长期记忆/总结

### 3.3 人工修改优先于 AI 原始草案
若人工对 AI 草案进行了改写，则后续系统应以改写后的版本为准，而不是原始草案。

### 3.4 可完全关闭自动化
系统必须支持纯人工模式：
- AI 仅建议
- AI 不自动推进
- AI 不自动写入历史
- AI 不自动改变人物行为模式

---

## 4. 审阅流程

### 4.1 标准流程
1. AI 生成草案
2. 系统将草案标记为 `pending_review`
3. 人工选择以下操作之一：
   - 通过（approve）
   - 编辑后通过（edit + approve）
   - 拒绝（reject）
   - 重新生成（regenerate）
   - 改为纯人工重写（manual replace）
4. 只有最终确认版本写入正式历史
5. 后续记忆、行为模式、状态变化基于最终确认版本更新

### 4.2 强制审阅项
以下内容必须人工确认后才能生效：
- [x] 关键剧情推进
- [x] 结局走向变化
- [x] 角色性格变化
- [x] 疯狂表现落实
- [x] 秘密信息揭示
- [x] 死亡/永久伤害/重大损失
- [x] 关键 NPC 态度变化
- [x] 其他：长记忆摘要、世界状态更新、角色关系重写

---

## 5. 审阅操作定义

### 5.1 Approve
含义：  
人工确认 AI 草案原样通过。

效果：
- 原草案成为正式记录
- 状态可据此更新
- 行为模式可据此学习

### 5.2 Edit and Approve
含义：  
人工修改 AI 草案后通过。

效果：
- 修改后的版本成为正式记录
- 原草案仅作审计保留，不作为后续依据
- 后续记忆与行为模型必须学习修改后的版本

### 5.3 Reject
含义：  
人工否决该草案。

效果：
- 不写入正式历史
- 不更新状态
- 不更新行为模式
- 可选择重新生成或改人工输入

### 5.4 Regenerate
含义：  
要求 AI 基于同一上下文重新给出草案。

效果：
- 原草案不生效
- 新草案重新进入审阅流程

### 5.5 Manual Replace
含义：  
人工完全重写，不采用 AI 草案。

效果：
- 人工版本成为正式记录
- AI 草案不作为正式依据
- 后续行为趋势默认参考人工版本，但允许在特殊配置下标记为“不要学习”

## 6. 草案与正式记录的数据要求

### 6.1 每个草案应包含
- draft_id
- actor_id
- actor_type
- visibility_scope
- draft_text
- structured_action
- rationale_summary
- created_at
- review_status

### 6.2 每个正式记录应包含
- final_id
- source_draft_id
- final_text
- final_structured_action
- approved_by
- approved_at
- review_decision
- supersedes_previous (如有)
- authoritative = true

以下内容在正式确认后，可用于更新 AI KP / AI 调查员后续行为：

- [x] 最终行动文本
- [x] 最终叙事风格
- [x] 人物行为偏好
- [x] 人物风险倾向
- [x] 与 NPC 的互动模式
- [x] 调查推进节奏

---

## 8. 行为学习限制

### 8.1 不允许学习的内容
以下内容即使出现在草案中，也不得直接作为行为学习依据：
- 被拒绝的草案
- 未确认的草案
- 被人工完全替换且标记为“不要学习”的内容
- 明显越界、跑题、失控内容

### 8.2 允许学习的内容
以下内容可作为后续条件输入：
- 已确认通过的最终文本
- 人工编辑后的最终文本
- 已确认的角色行为倾向
- 已确认的剧情方向选择

## 9. 审阅作用域 / 可见性

### 9.1 可见范围类型
- `public`：所有玩家可见
- `private_investigator`：仅特定调查员可见
- `kp_only`：仅 KP / 主持端可见
- `shared_subset`：部分调查员可见
- `system_internal`：仅系统内部使用

### 9.2 审阅权限
- 谁可以审阅 AI KP 输出：
  房主 / 主持人 / KP
- 谁可以审阅 AI 调查员输出：
  默认仅房主 / 主持人 / KP；测试模式下可允许对应玩家审阅自己的建议
- 谁可以审阅 NPC 私密输出：
  房主 / 主持人 / KP
- 是否允许玩家只审阅自己的私密建议：
  - [x] 是
  - [ ] 否

## 11. 审阅界面建议

- 生成者（AI KP / AI 调查员 / AI NPC）
- 作用域
- 草案文本
- 结构化动作摘要
- 生成理由摘要

- 编辑后通过
- 拒绝

另外建议显示：

- 是否会写入长期记忆

## 12. 特殊情况处理

### 12.1 时间敏感场景
在需要快速推进的场景下，是否允许采用“默认建议先显示，但仍待确认”的模式：
- [x] 是
- [ ] 否
- 说明：
  可先展示为建议文本，但在人工确认前不得写入正式历史或驱动后续状态更新。

### 12.2 连续小动作
对低风险、低影响动作是否允许简化审阅：
- [x] 是
- [ ] 否
- 允许范围：
  普通提问、非关键场景移动、低影响调查尝试、无秘密泄露风险的氛围对话。
  仍建议保留一键通过，而不是完全跳过。

### 12.3 高风险动作
以下动作必须强制人工确认：
- [x] 重大剧情转折
- [x] 角色死亡/残疾
- [x] 永久 SAN 重大变化
- [x] 大规模秘密公开
- [x] 结局锁定

## 13. 审计与日志
系统应记录：
- 原始草案
- 人工修改内容
- 最终通过版本
- 审阅者
- 时间戳
- 是否被后续版本覆盖

日志保留策略：
默认完整保留草案与最终版本，便于调试 AI 行为与排查信息泄漏。用户可在 campaign 级别关闭部分详细审计导出，但系统内部建议保留。

## 15. 当前已确认的审阅规则

### Review Rule RP-001
- 名称：AI 最终动作必须人工确认
- 适用对象：AI KP、AI 调查员、AI NPC
- 内容：任何会导致状态变化、正式文本记录、秘密变化、记忆更新的最终输出都必须人工确认
- 是否强制：是
- 示例：AI KP 提议“钟声响起，教堂地下入口打开”，必须经人工确认后才进入正式历史

### Review Rule RP-002
- 名称：人工编辑版本覆盖原草案
- 适用对象：所有 AI 草案
- 内容：人工编辑后的最终版本为唯一权威版本，后续记忆与行为建模必须以其为准
- 是否强制：是
- 示例：AI 调查员原本想“直接开枪”，人工改成“先观察后退”，系统后续应学习后者

### Review Rule RP-003
- 名称：高风险内容强制强审
- 适用对象：高风险草案
- 内容：涉及死亡、疯狂、秘密公开、结局、重大关系变化的草案必须明确人工确认，不得快速自动通过
- 是否强制：是
- 示例：AI 建议公开某调查员隐藏身份，必须人工确认
